## Prerequisites

Before you begin, make sure you have the following installed:

| Tool | Minimum version | Used for |
|---|---|---|
| Python | 3.10+ | FastAPI backend and ML model |
| Node.js | 18+ | React frontend build tooling |
| Git | any | Cloning the repository |

## Set up HealthIA

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Rafael-Lee1/healthia-diagnostico-inteligente.git
    cd healthia-diagnostico-inteligente
    ```
  </Step>

  <Step title="Install backend dependencies">
    Navigate to the `HEALTHIA/` directory and install the Python requirements.

    <CodeGroup>
      ```bash With virtual environment (recommended)
      python -m venv .venv
      source .venv/bin/activate   # On Windows: .venv\Scripts\activate
      pip install -r ../requirements.txt
      ```

      ```bash Without virtual environment
      pip install -r requirements.txt
      ```
    </CodeGroup>

    Key packages installed:

    | Package | Version | Purpose |
    |---|---|---|
    | fastapi | 0.131.0 | HTTP API framework |
    | uvicorn | 0.41.0 | ASGI server |
    | xgboost | 3.2.0 | ML classifier |
    | scikit-learn | 1.7.1 | TF-IDF, LabelEncoder |
    | pandas | 3.0.1 | Data handling |
  </Step>

  <Step title="Start the backend API">
    From inside the `HEALTHIA/` directory:

    ```bash
    uvicorn main:app --reload --port 8000
    ```

    Verify it's running by opening [http://127.0.0.1:8000](http://127.0.0.1:8000) — you should see:

    ```json
    {"Message": "Welcome to HealthIA API"}
    ```

    <Tip>
      Keep this terminal open. Open a second terminal window for the frontend setup.
    </Tip>
  </Step>

  <Step title="Install frontend dependencies">
    From the project root, navigate to the `frontend/` directory:

    <CodeGroup>
      ```bash npm
      cd frontend
      npm install
      ```

      ```bash yarn
      cd frontend
      yarn install
      ```

      ```bash pnpm
      cd frontend
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Configure the frontend environment">
    Create a `.env` file inside the `frontend/` directory:

    ```bash frontend/.env
    VITE_API_BASE_URL=http://127.0.0.1:8000
    ```

    <Note>
      If you deploy the backend to a different host or port, update `VITE_API_BASE_URL` to match.
    </Note>
  </Step>

  <Step title="Start the frontend dev server">
    From the `frontend/` directory:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    Open [http://localhost:5173](http://localhost:5173) in your browser. You should see the HealthIA interface with a green **API online** status badge in the header.
  </Step>
</Steps>

## Make your first prediction

### Using the UI

1. In the symptom input field, type symptoms separated by commas — for example: `febre, cansaço, dor no corpo`
2. You can also click on suggested symptom chips (febre, tremores, visão dupla, etc.) to add them quickly
3. Add at least **2 symptoms** (maximum 8 per analysis)
4. Click **Gerar diagnóstico**
5. The predicted condition appears on the right side of the screen

### Using curl

You can also call the API directly:

```bash
curl "http://127.0.0.1:8000/predict/?sintomas=febre,cansa%C3%A7o,dor%20no%20corpo"
```

Expected response:

```json
{
  "sintomas": ["febre", "cansaço", "dor no corpo"],
  "diagnostico_previsto": ["doenca_lyme"]
}
```

The `diagnostico_previsto` field contains the predicted condition label. See the [Supported Conditions](/conditions/overview) page for the full list of possible values and their English names.

<Warning>
  HealthIA provides educational predictions only. The results are **not** a substitute for professional medical diagnosis or advice. Always consult a qualified healthcare provider.
</Warning>
